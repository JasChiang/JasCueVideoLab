## API 原生座標規則

* 此 API response 使用 Gemini object-detection 原生順序。
* 原點位於左上角，x 向右，y 向下，全部正規化到 0–1000。
* 欄位名稱固定為 `box_2d_yxyx`，格式固定為 `[y_min, x_min, y_max, x_max]`。
* 必須滿足 `0 <= y_min < y_max <= 1000` 與 `0 <= x_min < x_max <= 1000`。
* 不得輸出 x-first 的 `box_2d` 欄位；本機 adapter 會在驗證後轉成 canonical `[x_min, y_min, x_max, y_max]`。
* 框應緊密包住指定目標，不含大量無關背景。
* 不得假設圖片外的內容，也不得以事件時間取代影像證據。

## 多個相似物件

若有多個合理候選，全部回傳，依 confidence 由高至低排序，並各自說明辨識理由。沒有足夠證據時不要任選。

## 遮擋與不可見

完全不可見時 visible=false、candidates=[]，並在 visibility_reason 說明原因，不得猜測座標。部分遮擋時 visible=true、occlusion=partial；若 bbox 含推定的完整範圍，須在 disambiguation_reason 明說並降低 confidence。

## 目標層級

嚴格遵守指定層級：手機螢幕不可框整支手機；手機不可框整條手臂；人物臉部不可框整個人；正在操作的手機不可選桌上另一支手機。

只回傳符合所提供 Structured Output schema 的資料。
