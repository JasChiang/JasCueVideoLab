## 座標規則

* 原點位於左上角，x 向右，y 向下，全部正規化到 0–1000。
* box_2d 固定為 [x_min, y_min, x_max, y_max]。
* 必須滿足 0 <= x_min < x_max <= 1000 與 0 <= y_min < y_max <= 1000。
* 框應緊密包住指定目標，不含大量無關背景。
* 不得假設圖片外的內容，也不得以事件時間取代影像證據。

## 多個相似物件

若有多個合理候選，全部回傳，依 confidence 由高至低排序，並各自說明辨識理由。沒有足夠證據時不要任選。

`match_status` 只描述指定實例是否匹配：唯一且可確認時用 `matched`；多個合理實例或身分無法區分時用 `ambiguous`；不存在、條件衝突或證據不足時分別用 `not_visible`、`target_mismatch`、`insufficient_evidence`，並回傳 `visible=false`、`candidates=[]`。

只有輸入含明確可觀察事件條件時才判斷 `predicate_status`，使用 `satisfied`、`not_satisfied` 或 `indeterminate`；沒有事件條件時使用 `not_applicable`。此欄位不得取代 target 身分、可見性或遮擋判斷。

## 遮擋與不可見

完全不可見時 visible=false、candidates=[]，並在 visibility_reason 說明原因，不得猜測座標。部分遮擋時 visible=true、occlusion=partial；若 bbox 含推定的完整範圍，須在 disambiguation_reason 明說並降低 confidence。

## 目標層級

嚴格遵守指定實例、層級與關係：局部不可框成整體，整體不可擴大到持有人或周邊；實體不可改成背景圖像或反射；帶有持有、操作、位置或狀態條件的 target 不可改選另一個同類實例。
